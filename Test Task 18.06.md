# Тестовое техническое задание — Sloth Exchange (мобильное приложение)

---

## 1. Общая информация

**Продукт:** Sloth Exchange — мобильное приложение для обмена валют.

**Цель задания:** разработать главный экран мобильного приложения по предоставленному дизайн-прототипу.

Инструкция по запуску проекта — в файле [SETUP.md](SETUP.md).

---

## 2. Бизнес-требования

Sloth Exchange — сервис P2P-обмена валют. Пользователь открывает приложение, видит актуальные курсы и может быстро рассчитать сумму обмена прямо на главном экране, после чего переходит к созданию сделки.

Главный экран выполняет три задачи:
- показывает пользователю актуальные курсы обмена;
- позволяет рассчитать сумму до создания заявки;
- является точкой входа во все ключевые разделы приложения.

---

## 3. Границы задания

**Входит в задание:**
- главный экран с калькулятором и таблицей курсов;
- переходы на экраны-заглушки по кнопкам;
- загрузка данных из мок-файлов (описаны в разделе 6).

**Не входит в задание:**
- реализация остальных экранов (заказы, профиль, чат, рефералы);
- авторизация и регистрация;
- реальная интеграция с backend API.

---

## 4. Функциональные требования

### 4.1. Шапка экрана

Необходимо отображать приветствие пользователя:
- если имя известно — «Привет, Иван!»
- если имя не известно — «Привет!»

Имя берётся из мок-файла `mock/GET_users_me.json`.

### 4.2. Блок быстрых действий

Необходимо реализовать четыре кнопки в ряд с иконками и подписями:

| Подпись | Экран назначения |
|---------|-----------------|
| Обменять | CreateOrderScreen |
| История | OrdersScreen |
| Поддержка | SupportScreen |
| Рефералы | ReferralScreen |

Все экраны назначения могут быть заглушками (пустой экран с названием).

### 4.3. Калькулятор обмена

Основной блок главного экрана. Данные берутся из `mock/GET_exchange-config.json` и `mock/GET_rates.json`.

**Поле «Отправляю»:**
- числовой ввод суммы;
- выбор валюты из списка уникальных `fromCurrency` (из `exchange-config`).

**Поле «Получаю»:**
- числовой ввод суммы;
- выбор валюты — только те `toCurrency`, которые соответствуют выбранной валюте отправки.

**Разделитель с кнопкой swap:**
- горизонтальная линия между полями;
- кнопка по центру — меняет местами валюты и суммы в полях;
- если обратная пара существует в `exchange-config` — она выбирается, иначе — берётся первая доступная пара для новой `fromCurrency`.

**Пересчёт в реальном времени:**
- при изменении любого поля второе пересчитывается автоматически;
- курс берётся из `mock/GET_rates.json` по полям `rateBase` и `rateQuote` активного конфига;
- курсы поддерживают тиры: ставка меняется при достижении минимальной суммы (`min`). Применяется наибольший подходящий тир;
- под полями отображается строка с текущим курсом: `1 CNY = 13.25 RUB`.

**Кнопка «Создать сделку»:**
- переход на CreateOrderScreen; (заглушка)
- кнопка неактивна, если курс не загружен или сумма равна нулю.

### 4.4. Список курсов

Под калькулятором — список всех курсов из `mock/GET_rates.json`.

Каждая строка содержит:
- пару валют: `RUB / CNY`;
- текущий курс (значение первого тира);

Бейджи изменений в % реализовывать не нужно — достаточно визуальной заглушки (статичный бейдж с любым значением).

Над списком — три таба фильтрации:
- **Все** — все курсы;
- **Фиат** — пары, где `base` не является криптовалютой;
- **Крипто** — пары, где `base` или `quote` = USDT.

## 5. Нефункциональные требования

### 5.1. Стек

Обязателен следующий стек:

- **Expo SDK 56** — стартовый проект уже инициализирован в `apps/mobile/`.
- **TypeScript** — входит в шаблон по умолчанию.

## 6. Mock-данные

Вместо реального API необходимо использовать локальные JSON-файлы. Файлы лежат в `apps/mobile/mock/`.

| Файл | Эмулирует endpoint | Содержимое |
|------|--------------------|------------|
| `mock/GET_users_me.json` | `GET /api/users/me` | Профиль текущего пользователя |
| `mock/GET_exchange-config.json` | `GET /api/exchange-config` | Список доступных валютных пар |
| `mock/GET_rates.json` | `GET /api/rates` | Актуальные курсы с тирами |

Данные необходимо читать при старте приложения. Структура ответов соответствует реальному API — замена моков на сетевые запросы должна требовать минимальных изменений.

В репозитории также есть готовый `apps/api/` — NestJS-сервер, который отдаёт те же данные через HTTP (`GET /api/...`). Его можно поднять через Docker (см. SETUP.md) и переключить мобильное приложение с файловых моков на сетевые запросы — это не обязательно, но приветствуется.

## 7. Критерии оценки

**Обязательно:**
- логика пересчёта курса работает корректно (тиры, двустороннее редактирование, swap);
- список курсов фильтруется по табам;
- код читаем.
