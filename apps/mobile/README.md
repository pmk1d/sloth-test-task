# Sloth Exchange - мобильное приложение (главный экран)

Стек: **Expo SDK 56**, React Native 0.85, TypeScript (strict), expo-router.

## Запуск

```bash
# из корня репозитория
npm install
cd apps/mobile
npx expo start            # откройте в Expo Go или dev-build
```

По умолчанию данные читаются из локальных моков (`mock/GET_*.json`).
Чтобы переключиться на реальный API из `apps/api` - задайте переменную и всё:

```bash
EXPO_PUBLIC_API_URL=http://localhost:3001 npx expo start
```

Формат моков и HTTP-ответов идентичен, поэтому переключение не требует правок
кода (см. `src/api/client.ts`).

### Логика калькулятора (`src/lib/exchange.ts`)

Ключевые решения (математика - главный критерий оценки):

- **Курс**: `1 BASE = rate QUOTE` (например `1 CNY = 13.25 RUB`).
- **Тиры считаются в QUOTE-валюте (RUB)** - в валюте расчётов. Один и тот же
  `rate_001` (CNY/RUB) используется и для `RUB→CNY` (cfg_001), и для `CNY→RUB`
  (cfg_004); RUB - единственная общая «фиксированная» сторона, а пороги
  `10 000 / 50 000 / 100 000` имеют смысл только как рублёвые суммы.
- **Наибольший подходящий тир** - `tierForQuoteAmount`.
- **Двустороннее редактирование**: правка поля QUOTE → тир напрямую; правка поля
  BASE → решаем обратную задачу (`solveFromBase`), выбирая наибольший тир, чей
  порог выполняется. Для монотонных курсов это даёт точный round-trip
  (`from → to → from` стабильно), что проверено тестами.
- **Swap**: берём обратную пару из конфига, иначе первую доступную пару для новой
  `fromCurrency`; если такой нет - свап недоступен (кнопка неактивна), вместо
  перехода в невалидное состояние.

Проверка математики на реальных моках (27 ассертов):

```bash
node --experimental-strip-types src/lib/exchange.verify.ts
```

### Дизайн vs данные

Референс (`design/Main Page.png`) - источник правды по **внешнему виду**, моки -
по **данным**. Они намеренно расходятся (в референсе нарисованы пары вроде
USDT→CNY, которых нет в конфиге). Экран отрисован «точь-в-точь» по референсу, но
все числа/пары берутся из моков. Бейджи `%` в списке курсов - статичные
плейсхолдеры (детерминированы по `id`, как и требует ТЗ).

### Сознательные отклонения от ТЗ

- **Список курсов показывает полный `quote` из БД** (`RUB-TBANK /CNY`,
  `RUB-BANKS /CNY`), а не сокращённый `RUB / CNY` из примера §4.4. Причина:
  `rate_001` и `rate_002` оба имеют `base = CNY` и при сокращении до `RUB`
  давали бы **две визуально одинаковые строки** `RUB / CNY`. Рельс (`-TBANK` /
  `-BANKS`) их различает. Математика и фильтры табов не затронуты.
- **Подписи «Все валюты» и «Курс:»** взяты из Figma-референса (там именно так),
  хотя текст ТЗ §4.4/§4.3 пишет «Все» и пример без префикса «Курс:». Приоритет
  отдан дизайну, как и требовалось («точь-в-точь как в референсе»).
- **Тир `min` трактуется в QUOTE-валюте (RUB)** - единственное связное чтение:
  все курсы котируются в RUB, а `rate_001` (CNY/RUB) используется и для
  `RUB→CNY`, и для `CNY→RUB`, так что общая «фиксированная» сторона - RUB.

### Ассеты

Точные SVG из Figma лежат в `src/icons/svg.ts` и рисуются 1:1 через
`react-native-svg`: иконки быстрых действий, шестерёнка и монеты
(RUB/CNY/KZT/UZS/VND/THB/USDT). Монеты RUB в Figma нет, поэтому `coinRUB`
дорисован вручную в том же стиле. Текстовый фолбэк в `CurrencyBadge` нужен только
для валюты, которой нет в `COIN_SVG` (в текущих данных не встречается). Мелкие
глифы интерфейса (шевроны, галочка, стрелка свапа) - из `@expo/vector-icons`.
